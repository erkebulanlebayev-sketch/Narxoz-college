# Управление пользователями

## Что сделано

✅ Создана страница `/admin/users` для управления всеми пользователями
✅ Показываются все зарегистрированные студенты и учителя
✅ Разделение по ролям с фильтрацией (Все / Студенты / Учителя)
✅ Статистика: общее количество, студентов, учителей
✅ Real-time синхронизация изменений
✅ Возможность удаления пользователей
✅ Современный дизайн с градиентами

## Как использовать

### 1. Создать таблицы в Supabase (если их нет)

Откройте Supabase SQL Editor и выполните файл `create-users-tables.sql`:

```sql
-- Этот скрипт создаст таблицы students и teachers
-- с правильными RLS политиками и Real-time
```

### 2. Открыть страницу управления

1. Войдите как администратор
2. Перейдите в раздел "Пользователи" или откройте `/admin/users`
3. Вы увидите список всех зарегистрированных пользователей

## Функции страницы

### Статистика
- **Всего пользователей** - общее количество студентов и учителей
- **Студентов** - количество зарегистрированных студентов
- **Учителей** - количество зарегистрированных учителей

### Фильтрация
- **Все** - показать всех пользователей (студенты + учителя)
- **Студенты** - показать только студентов
- **Учителя** - показать только учителей

### Информация о пользователях

#### Для студентов:
- Имя
- Email
- Группа (ИС-21-1, ИС-21-2, и т.д.)
- GPA (средний балл)
- Дата регистрации

#### Для учителей:
- Имя
- Email
- Предмет
- Дата регистрации

### Действия
- **Удалить** - удалить пользователя из системы (с подтверждением)

## Как работает регистрация

### Студенты
1. При регистрации студент выбирает группу
2. Данные сохраняются в `auth.users` (email, пароль)
3. Метаданные сохраняются: имя, роль, группа
4. Создаётся запись в таблице `students` с группой и GPA

### Учителя
1. При регистрации указывается роль "teacher"
2. Данные сохраняются в `auth.users`
3. Создаётся запись в таблице `teachers`
4. Предмет можно указать позже

## Real-time синхронизация

Страница автоматически обновляется при:
- Добавлении нового студента
- Добавлении нового учителя
- Удалении пользователя
- Изменении данных

## Структура таблиц

### students
```sql
- id (BIGSERIAL PRIMARY KEY)
- name (TEXT NOT NULL)
- email (TEXT UNIQUE NOT NULL)
- group_name (TEXT NOT NULL)
- gpa (DECIMAL(3,2) DEFAULT 0.00)
- attendance (INTEGER DEFAULT 0)
- created_at (TIMESTAMPTZ)
- updated_at (TIMESTAMPTZ)
```

### teachers
```sql
- id (BIGSERIAL PRIMARY KEY)
- name (TEXT NOT NULL)
- email (TEXT UNIQUE NOT NULL)
- subject (TEXT DEFAULT '')
- created_at (TIMESTAMPTZ)
- updated_at (TIMESTAMPTZ)
```

## Безопасность (RLS)

### Students
- **SELECT**: Все могут читать
- **INSERT**: Админы и студенты (для регистрации)
- **UPDATE**: Только админы
- **DELETE**: Только админы

### Teachers
- **SELECT**: Все могут читать
- **INSERT**: Админы и учителя (для регистрации)
- **UPDATE**: Только админы
- **DELETE**: Только админы

## Тестирование

1. Зарегистрируйте нового студента через `/register`
2. Зарегистрируйте нового учителя
3. Войдите как администратор
4. Откройте `/admin/users`
5. Проверьте, что все пользователи отображаются
6. Попробуйте фильтры (Все / Студенты / Учителя)
7. Проверьте удаление пользователя
