## Создание тестовых аккаунтов

### Способ 1: Через регистрацию (Рекомендуется)

1. Перейдите на `/register`
2. Зарегистрируйте каждый аккаунт с указанными выше данными
3. Для администратора:
   - После регистрации откройте Supabase Dashboard
   - Перейдите в Authentication > Users
   - Найдите пользователя `admin@narxoz.kz`
   - Откройте его профиль
   - В разделе "User Metadata" измените на:
     ```json
     {
       "name": "Администратор Системы",
       "role": "admin"
     }
     ```

### Способ 2: Через Supabase Dashboard

1. Откройте Supabase Dashboard
2. Перейдите в Authentication > Users
3. Нажмите "Add user" > "Create new user"
4. Введите email и пароль
5. После создания добавьте User Metadata:
   ```json
   {
     "name": "Имя пользователя",
     "role": "admin" | "teacher" | "student"
   }
   ```

## Оптимизации производительности

Система оптимизирована для быстрой работы:

### 1. Dynamic Imports
- Компоненты Carousel и PillNav загружаются динамически
- Уменьшает начальный размер бандла
- Показывает placeholder во время загрузки

### 2. Мемоизация
- `useMemo` для статических данных (меню, карусель)
- Предотвращает лишние перерендеры
- Улучшает производительность навигации

### 3. Оптимизация изображений
- Используется Next.js Image для автоматической оптимизации
- Lazy loading для изображений вне viewport

### 4. Code Splitting
- Автоматическое разделение кода по роутам
- Каждая страница загружается отдельно

## Известные особенности

- При первом входе может быть небольшая задержка (инициализация Supabase)
- Последующие переходы между страницами мгновенные
- Carousel загружается с небольшой задержкой (динамический импорт)
