## Установка

### Шаг 1: Выполнить SQL скрипт

Откройте Supabase Dashboard → SQL Editor и выполните:

```bash
supabase-enable-all-realtime.sql
```

Этот скрипт включит real-time для всех таблиц.

### Шаг 2: Проверка

После выполнения скрипта вы увидите список всех таблиц с real-time:
- news ✅
- schedule ✅
- shop_products ✅
- grades ✅
- library_books ✅
- materials ✅
- settings ✅
- students ✅
- teachers ✅

### Шаг 3: Тестирование

1. Откройте 2 браузера (или 2 вкладки в режиме инкогнито)
2. В первом войдите как админ
3. Во втором войдите как студент/преподаватель
4. Создайте новость в админке
5. Новость появится у студента/преподавателя мгновенно! 🎉

## Как это работает

### Real-Time подписка
```typescript
const channel = supabase
  .channel('unique-name')
  .on('postgres_changes', 
    { event: '*', schema: 'public', table: 'table_name' },
    () => loadData()
  )
  .subscribe();
```

### Fallback Polling
```typescript
const interval = setInterval(loadData, 10000);
```

### Cleanup
```typescript
return () => {
  supabase.removeChannel(channel);
  clearInterval(interval);
};
```

## Производительность

- **Latency**: < 100ms
- **Bandwidth**: Минимальный
- **Scalability**: Тысячи подключений
- **Reliability**: Fallback polling если Realtime не работает

## Безопасность

- RLS политики контролируют доступ
- Студенты видят только свои данные
- Преподаватели видят только свои группы
- Админы видят все
