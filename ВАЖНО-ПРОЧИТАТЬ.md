# 📋 Важные файлы и инструкции

## 🔧 Файлы для исправления ошибок

### 1. CHECK-AND-FIX-SCHEDULE-FINAL.sql
**Назначение**: Исправление таблицы schedule
**Когда использовать**: Если есть ошибки при добавлении расписания
**Что делает**:
- Показывает структуру таблицы schedule
- Удаляет лишние колонки (time, day_of_week)
- Убирает NOT NULL constraints
- Исправляет все несоответствия

### 2. ПОЛНОЕ-ИСПРАВЛЕНИЕ-RLS.sql
**Назначение**: Исправление RLS политик
**Когда использовать**: Если есть ошибка "permission denied for table users"
**Что делает**:
- Удаляет все старые RLS политики
- Создаёт новые правильные политики с auth.jwt()
- Исправляет доступ для всех таблиц

### 3. supabase-enable-all-realtime.sql
**Назначение**: Включение real-time синхронизации
**Когда использовать**: Если изменения не синхронизируются между пользователями
**Что делает**:
- Включает real-time для всех таблиц
- Проверяет существование таблиц
- Показывает список таблиц с real-time

## 🚀 Быстрый старт

### Если есть ошибки:

1. **Ошибка с расписанием** → Выполните `CHECK-AND-FIX-SCHEDULE-FINAL.sql`
2. **Ошибка permissions** → Выполните `ПОЛНОЕ-ИСПРАВЛЕНИЕ-RLS.sql`
3. **Нет синхронизации** → Выполните `supabase-enable-all-realtime.sql`
